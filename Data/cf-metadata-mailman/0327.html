The above message contained the following:

> ... The udunits library has some
> limitations (e.g. 365_day_calendar isn't supported),

That's correct. Time-offset units were added to the UDUNITS package
in order to provide for time units a capability similar to what the
celsius unit provides for temperature.

It was a mistake. User's have tried to use this feature to do what
should be done, instead, by a package that supports physical quantities.
The physical quantities package could use the UDUNITS package, but the
UDUNITS package, itself, was never intended or designed to support
the manipulation of physical quantities -- only units of physical
quantities.
