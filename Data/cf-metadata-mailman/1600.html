Dear Roy,

This all looks good and the availability of the mappings is excellent.
It might be worth reminding everyone that more than one alias can map to
the same current name, e.g. the quantity we currently call
lagrangian_tendency_of_air_pressure has had two previous incarnations:
omega and vertical_air_velocity_expressed_as_tendency_of_pressure. Both
of these appear in your list of mappings.

Best wishes,
Alison

> Dear All,
>
> The Standard names are now served from the vocabulary server as three
> lists:
>
> http://vocab.ndg.nerc.ac.uk/list/P07/9 contains all the Standard Names
> that have ever been published, including names that have been aliased
>
> http://vocab.ndg.nerc.ac.uk/list/P071/9 contains the Standard Names
> that are currently valid (i.e. those that have not been deprecated
> through aliasing)
>
> http://vocab.ndg.nerc.ac.uk/list/P072/9 contains the Standard Names
> that have been deprecated through aliasing
>
> The mapping between deprecates and their replacement aliases my be
> obtained through the following API call
>
>
http://vocab.ndg.nerc.ac.uk/axis2/services/vocab/getMap?subjectList=htt
>
p://vocab.ndg.nerc.ac.uk/list/P072/current&predicate=255&objectList=htt
> p://vocab.ndg.nerc.ac.uk/list/P071/current&inference=false
>
> Note that the P072 entryTermLastMod element gives the date of
> deprecation for terms aliased since version 2 of the Standard Names.
> Otherwise it is set to the date when the system first served Standard
> Names (2006-09-26).
>
