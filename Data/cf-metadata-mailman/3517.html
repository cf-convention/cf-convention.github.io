Hi Glenn,

On 26/05/11 16:11, Comiskey, Glenn wrote:
> Having recently taken up an appointment that involves me working with
> meteorological/ocenographic datasets I have been reading the CF
> Conventions documentation. I've noted that the documention, when
> refering to the units variable attribute, references UNIDATA's udunits
> package and the data file udunits.dat. This data file appears to
> relate specifically to v1.12.11 of the udunits package, while I note
> that the current release is v2.1.23. Does the CF Convention only
> recognise unit values listed in v1.12.11 or is it meant to encompass
> v2.1.23 as well.

There was some debate about this last year - see the threads in the
archive with subjects containing "udunits 1 or 2 for CF". There are
certainly differences in interpretation of units that caused some
problems. The following link gives you a starting point, but note the
threading was broken a number of times:
 http://mailman.cgd.ucar.edu/pipermail/cf-metadata/2010/thread.html#6924

I think the discussion was reasonably conclusive that a) one use full
names (i.e. milligram rather than mg) and prefixes to avoid some
differences in interpretation and, much more tentatively, b) that CF
should recommend udunits2. However, that doesn't seem to have been
enacted, so probably I missed something later!

There was also a big discussion on time specifications later on, which
suggested not using udunits for time. I'm not sure that a firm
conclusion was reached.

Cheers,

Mike.
