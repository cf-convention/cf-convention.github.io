Have just upgraded ERDDAP to v1.36 and attempted to add a new GRIB file
and note that when I run GenerateDatasetsXml.bat the dataset XML
generated includes the line <att name="CF:feature_type">GRID</att>,
listed as a source attribute. Take it this is meant to relate to CF
Discrete Sampling Geometries convention, but seems odd that attribute
value is not valid, i.e. I believe only point, timeSeries, trajectory,
profile, timeSeriesProfile and trajectoryProfile are valid, and the
attribute name is inconsistent with the the convention which only refers
to attribute name "featureType".
 
Can someone detail what what convention "CF:feature_type" is supposedly
from.
