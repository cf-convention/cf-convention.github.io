Regarding Glenn Comiskey's email about CF:feature_type:

The CF:feature_type attribute was in one of the earlier, draft versions
of the CF Point Observation Conventions
(https://cf-pcmdi.llnl.gov/trac/wiki/PointObservationConventions), which
is now called Discrete Sampling Geometries. But that attribute name was
rejected in later versions because the ":" in the attribute name isn't
allowed and caused problems.

It seems likely that the version of netcdf-java (perhaps now
out-of-date), which is in that version of ERDDAP, generated that
attribute based on that earlier draft version of the standard.

The work-around for ERDDAP is probably:
* Set the value of CF:feature_type to "null".
* Set cdm_data_type to the desired feature type (from ERDDAP's list of
allowed values, which should be the same as CF's current list). If that
doesn't solve the problem, please contact me directly (not via this list).
