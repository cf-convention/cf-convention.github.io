Le 21/03/2012 20:59, alison.pamment at stfc.ac.uk a ?crit :
> Dear All,
>
> The standard name table will be updated tomorrow with new names, updated definitions and units that have been agreed over recent months. Details of the planned changes are given at the end of this message. The changes will be submitted to the NERC vocabulary server today and should be published overnight. Tomorrow I will update the xml and html versions of the standard name table on the CF website. In this way the date and version information in the two locations will be kept synchronised. I will post again when the update is complete.
>
> A number of proposals are still under discussion and these will NOT be included in the update tomorrow. Many of them are quite close to agreement, but I have to draw the line somewhere in order to finalise the list of names and definitions that will go into the table this time around. I will make another update to the standard name table in one month's time and names that are agreed in the intervening period will be added at that stage. The names that are not yet finalised are:
>
> 1) volume_extinction_coefficient_in_air_due_to_mie_scattering_of_ambient_aerosol; m-1 (proposed by Martin Schultz).
> There is still some discussion centred on extinction/scattering/attenuation terminology. See http://mailman.cgd.ucar.edu/pipermail/cf-metadata/2012/027812.html for latest posting.
>
> 2) Emission names (proposed by Martin Schultz).
> We have now agreed on the syntax of these names and the new chemical species to be added. There has been some discussion of the names and definitions of emissions sectors. I want to allow a few more days to finalise this discussion, but I think we are nearing agreement on the sectors originally proposed by Martin. See http://mailman.cgd.ucar.edu/pipermail/cf-metadata/2012/027814.html for latest posting.
>
> 3) New surface aerosol optical properties names (proposed by Markus Fiebig).
> See trac ticket 81 for proposal and http://mailman.cgd.ucar.edu/pipermail/cf-metadata/2012/027835.html for latest posting.
>
> Best wishes,
> Alison
>

>

> mass_concentration_of_pm2p5_ambient_aerosol_in_air; kg m-3

>

>
> 2. Changes to definitions of existing names
>
> a. Resulting from TEOS-10 discussion.
>
> Give a proper definition of sea_water_salinity and make the use of the units clear. Proposed by Nan Galbraith.
>
> Update all existing salinity name definitions to emphasize that "salinity" is generic (consistent with updated sea_water_salinity definition). Proposed by Alison Pamment. This affects the following existing names:
> change_over_time_in_sea_water_salinity; 1 e-3
> sea_surface_salinity; 1 e-3
> sea_ice_salinity; 1e-3
> tendency_of_sea_water_salinity; 1 e-3 s-1
> tendency_of_sea_water_salinity_due_to_advection; 1 e-3 s-1
> tendency_of_sea_water_salinity_due_to_bolus_advection; 1 e-3 s-1
> tendency_of_sea_water_salinity_due_to_vertical_mixing; 1 e-3 s-1
> tendency_of_sea_water_salinity_due_to_sea_ice_thermodynamics; 1 e-3 s-1
> tendency_of_sea_water_salinity_due_to_horizontal_mixing; 1 e-3 s-1
> product_of_northward_sea_water_velocity_and_salinity; m s-1
> product_of_eastward_sea_water_velocity_and_salinity; m s-1
>
> Create a definition for sea_water_temperature (previously one did not exist). Update all other sea_water_temperature name definitions to included the same information. Proposed by Jonathan Gregory and Alison Pamment. This affects the following existing names:
> sea_water_temperature; K
> tendency_of_sea_water_temperature; K s-1
> tendency_of_sea_water_temperature_due_to_vertical_mixing; K s-1
> tendency_of_sea_water_temperature_due_to_horizontal_mixing; K s-1
> tendency_of_sea_water_temperature_due_to_bolus_advection; K s-1
> tendency_of_sea_water_temperature_due_to_advection; K s-1

> integral_of_sea_water_temperature_wrt_depth_in_ocean_layer; K m
> change_over_time_in_sea_water_temperature; K
>
> 3. Changes to units of existing names
>
> a. Resulting from TEOS-10 discussion.
>
> In addition to updating its definition to emphasise generic salinity, the units of
> sea_ice_salinity
> will be changed from psu to 1 e-3. This is because psu is not a unit in its own right.
