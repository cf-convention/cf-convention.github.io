On 03/20/2013 04:51 PM, John Caron wrote:

> I guess the point is that its not always fair to assume that, and the
> user wont know when it fails, esp for
>
> '2434567 days since -4713-01-01T00:00:00Z'
>
> unless she also computes
>
> '1952-08-15T00:00:00Z'
>
> which presumably she could do as a double check.
>
>
> Which library would you use? Does it work for non-standard calendars?
As I see it this should be something that can be done with ncdump (-t),
which should be a fairly standard tool available to data writers. If
ncdump misses a feature to support the proper calendar for conversion,
then it should be fixed there, maybe.
Of course it is never a mistake if other libraries/programs support this
conversion well.
But anyway, it would be a conversion to be handled by a library and
won't have to be encoded into the "raw" file. I think this is David's
point as well.
