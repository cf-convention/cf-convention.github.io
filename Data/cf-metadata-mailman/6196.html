Dear Randy

> We have data variables containing observation data that have corresponding flag variables containing status info (i.e. status_flag). These flag variables are ancillary data and the observation data variable includes the ancillary_variables attribute.
>
> The observation data variable makes use of coordinates variable for auxiliary coordinate variables, and includes the coordinates attribute. There are dependencies between values assigned to the auxiliary coordinate variables, and the values assigned to the ancillary variable (in this case a flag variable)
>
> Should the coordinates attribute be included in the specification of the ancillary variable ?

Yes, I think so. An ancillary variable is a data variable, and it should be
self-describing in its own right. In particular, it should have all the
coordinates which are relevant to it.

Best wishes

Jonathan
