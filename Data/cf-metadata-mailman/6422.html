Hi Nan, great question. I probably caused that minor snafu so I'll reply. Using the trac system is fine and still encouraged, to my knowledge. It is especially valuable for policy issues requiring discussion in the CF community.

I found it helpful to track details and subtleties of the web site in the GitHub issues tracker*, because I could create a bunch of individual issues, track each separately, and relate them to the corresponding fixes, without filling up the trac system with 'developer' issues that no one need worry about. I might have jumped the shark using the Github system so much, but Matthew was nice enough to go with it.

* not 'Google issues tracker' as I last wrote, sorry!!
