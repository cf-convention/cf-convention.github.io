> Why emulate separate repos when you can have the real thing?


I suppose the reason would be not so much one repo as one gitHub project --
so they would share the issue system, etc.

I think that the name list and conventions doc are really quite distinct,
so it would actually be _better_ to have completely separate projects in
this case.

For example, I'm imagining that CF2.* would share the same standard name
list as CF 1.*, for example. The name list really is its own "thing".
