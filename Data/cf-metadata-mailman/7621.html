Dear All,

The standard name and area_type tables have both been updated with the changes listed at the end of this message.

The current version of the standard name table is now V33, dated 17 May 2016. The current version of the area_type table is now V4, also dated 17 May 2016. The links on the CF website have been updated and the following new names have been added to the list of standard name contributors: Kristopher Bedka, Gerben de Boer, Elodie Fernandez, Oystein Godoy, Birgit Hassler, Reyna Jenkyns, Eizi Toyoda. The other contributors to this update were already listed.

The changes to standard names and area types have also been published on the NERC vocabulary server (standard names is vocabulary list P07 - http://vocab.nerc.ac.uk/collection/P07/current/; area_types is vocabulary list P29 - http://vocab.nerc.ac.uk/collection/P29/current/).

The next update to standard names will take place on 13th June.

Best wishes,
Alison

=== Standard name changes ===

1. New names

a. CMIP6 (OMIP) names proposed by Jonathan Gregory and Stephen Griffies

net_rate_of_absorption_of_shortwave_energy_in_ocean_layer
sea_water_potential_temperature_at_sea_floor

b. CDOM name proposed by John Maurer

concentration_of_colored_dissolved_organic_matter_in_sea_water_expressed_as_equivalent_mass_fraction_of_quinine_sulfate_dihydrate

c. depth_below_sea_floor proposed by Nan Galbraith and Reyna Jenkyns

d. Infrafred cloud top names proposed by Kristopher Bedka

air_temperature_at_effective_cloud_top_defined_by_infrared_radiation
height_at_effective_cloud_top_defined_by_infrared_radiation
pressure_at_effective_cloud_top_defined_by_infrared_radiation

e. UV index names proposed by Oystein Godoy

ultraviolet_index
ultraviolet_index_assuming_clear_sky
ultraviolet_index_assuming_overcast_sky

f. Liquid and Ice cloud fractions; cloud particle extinction. Proposed by Alejandro Bodas-Salcedo.

ice_cloud_area_fraction
ice_cloud_area_fraction_in_atmosphere_layer
liquid_water_cloud_area_fraction
liquid_water_cloud_area_fraction_in_atmosphere_layer
volume_extinction_coefficient_in_air_due_to_cloud_particles

g. Waves "from" names proposed by Gerben de Boer

sea_surface_swell_wave_from_direction
sea_surface_wind_wave_from_direction

h. Copernicus Marine Services name proposed by Elodie Fernandez

mole_concentration_of_zooplankton_expressed_as_nitrogen_in_sea_water

i. Flood water simulation names proposed by Eizi Toyoda

eastward_flood_water_velocity
northward_flood_water_velocity
flood_water_speed
time_of_maximum_flood_depth
time_when_flood_water_falls_below_threshold
time_when_flood_water_rises_above_threshold
flood_water_duration_above_threshold
flood_water_thickness
ground_level_altitude

j. River discharge name proposed by Rich Signell

water_volume_transport_in_river_channel

k. Ozone name proposed by Birgit Hassler

number_concentration_of_ozone_molecules_in_air

l. Standard name appearing in Appendix D of CF conventions but missing from standard name table - reported by Martin Juckes

ocean_sigma_z_coordinate

2. Modifications to definitions of existing names

a. High, medium, low type cloud names. Modification proposed by Mark Hedley.

The definitions of the following three names were modified to contain the sentence 'X_type_cloud_area_fraction is generally determined on the basis of cloud type, though Numerical Weather Prediction (NWP) models often calculate them based on the vertical location of the cloud.' This is to clarify their use with model data as well as observations.
high_type_cloud_area_fraction
medium_type_cloud_area_fraction
low_type_cloud_area_fraction

3. Modifications to units of existing names

The units of tendency_of_air_temperature_due_to_radiative_heating (K s-1) were missing from the standard name table - this was reported by Martin Juckes. The units have now been added.

4. Aliases

No new aliases are included in this update

=== Area type changes ===

1. New area types

a. Sea ice melt pond types proposed by Thomas Lavergne

sea_ice_melt_pond: 'The area classified as type "sea_ice" within a grid cell is unaltered when an area of type "sea_ice_melt_pond" is also present because melt ponds occur on top of the existing sea ice.'

melt_pond_free_sea_ice: 'The area occupied by type "sea_ice" is the sum of the areas of types "sea_ice_melt_pond" and "melt_pond_free_sea_ice".'

2. Modification to definitions of existing area types

a. Two definitions were added to clarify the relationship between the area_types sea, sea_ice, sea_ice_melt_pond and melt_pond_free_sea_ice. These changes were proposed by Thomas Lavergne.

sea: 'The area occupied by type "sea" is equal to the sum of the areas of types "ice_free_sea" and "sea_ice".'

sea_ice: 'The area occupied by type "sea_ice" within a grid cell is the sum of the areas of types "sea_ice_melt_pond" and "melt_pond_free_sea_ice". Melt ponds occur on top of the existing sea ice. The area occupied by type "sea" is equal to the sum of the areas of types "ice_free_sea" and "sea_ice".'

Previously no definitions were given for these two area types.

=== End of changes ===
