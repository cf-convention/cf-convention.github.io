Dear All,

The standard name table was updated on 7th August to version 58 and has now been updated again to version 59, dated 25th September 2018. Details of both updates are listed at the end of this message. All changes have also been published in the NERC Vocabulary Server.

Andrew Saulter has been added to the list of standard name table contributors: http://cfconventions.org/Data/cf-standard-names/docs/standard-name-contributors.html.

The September update contains just a few names. However, there has been good progress in the discussion of the platform orientation names (yaw, pitch, etc.) and I expect we will be able to finalize them in time for the next update, planned for 16th October.

Best wishes,
Alison

3. Modifications to definitions of existing names
a. Platform names
In the discussion of the new platform_heave names it was agreed that the definition of 'platform' should be expanded to read 'Platforms include, but are not limited to, satellites, aeroplanes, ships, instruments and buoys'. This text was added to the definitions of 11 existing names:

c. The name water_evaporation_amount_from_canopy has an alias of water water_evaporation_amount_from_canopy_where_land. Text was added to the definition to explain the deprecation of 'where' names: 'Unless indicated in the cell_methods attribute, a quantity is assumed to apply to the whole area of each horizontal grid box. Previously, the qualifier "where_type" was used to specify that the quantity applies only to the part of the grid box of the named type. Names containing the where_type qualifier are deprecated and newly created data should use the cell_methods attribute to indicate the horizontal area to which the quantity applies.'
