Agenda/Notes

  • Update/discussion on re-org of standard names (all vocabularies) repository
    • Current page (test): https://cfconventions.org/vocabularies/
    • Unfortunately, XSLT being removed from browsers for security and aging technology reasons.
    • Implement a build process - issue # 276
  • How to implement groups of equivalent standard names for sea and inland water: Vocab issue #25
    • We lack a semantically clear way of expressing this in the current standard name XML, aliases currently have an implicit meaning of “deprecated”.
    • Full adoption of NVS as canonical CF standard name source, specifically its owl:sameAs mapping, would allow non “deprecated” equivalent names to exist simultaneously with a clear semantic relationship.
      • How to map this relationship back into the CF Standard Name XML would still remain if the XML continues to be desired.
      • A conversation would need to be had with NVS about this (Gwen)
    • The discussion, however, was dominated by debate over the wisdom of this change.
