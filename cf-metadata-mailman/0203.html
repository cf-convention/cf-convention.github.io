Hi Alison,

Welcome back!

Right, what I want to do is to generate an ontology containing the mapping between the BODC Parameter Discovery Vocabulary, the CF Standard Names and the GCMD Science Keywords (that used to be called Parameter Valids). This is going to be 'for real' and therefore needs to be maintainable, which implies that it is built from versioned vocabularies and that the differences between the versions mapped and any subsequent versions need to be accurately documented in detail so that I can successfully manage ontology maintenance.

The way I would like to do this in practice would be to add current synchronised copies of CF Standard Names and the GCMD Science Keywords to the BODC-maintained NERC DataGrid Vocabulary Server. This is an automatically managed back end vocabulary storage system maintaining creation/modified timestamps for all terms, versioning for all term lists and an audit trail of all changes made. The back end is interfaced by a Web Service API (http://www.bodc.ac.uk/products/web_services/vocab/). A demonstration client (http://vocab.ndg.nerc.ac.uk/client/vocabServer.jsp) using the Web Service API is temporarily available until the appropriate part of the BODC web site has been redeveloped to use the API.

If this is done, I will get Michael Hughes in BODC to extend Luis Bermudez's voc2OWL software to output OWL files from our vocabulary back end (an Oracle database). This will be used to output the lists to be mapped and I will then use the VINE tool to actually do the mapping.

For this to happen, I need the following.

>From CF:

The blessing of the CF community to serve Standard Names from the NDG server (would NOT replace existing sources as canonical units not served and existing synonym management not carried through)
A spreadsheet containing two columns - standard name and its accompanying description (standard name version zero, which increments by 1 for each subsequent maintenance session)
Update spreadsheets for new standard names
Information on any changes/corrections to existing standard names or their descriptions

>From GCMD:

The blessing of GCMD to serve Science Keywords operationally (a demonstration snapshot is there at the moment)
Agreement on start point and maintenance mechanism (MEDI meeting next week gives us an opportunity)
Assistance upgrading our demonstration snapshot to any agreed start point

Here's hoping we can make this happen.

Cheers, Roy.
