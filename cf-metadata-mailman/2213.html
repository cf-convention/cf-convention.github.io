Good afternoon,

The SeaRISE (http://websrv.cs.umt.edu/isis/index.php/
SeaRISE_Assessment) project is committed to following CF Metadata
conventions in the model output files (at least from the whole ice-
sheet models), and we need to extend the standard name table to
include more fields used in glaciology and ice-sheet modeling.

I am creating a wiki-page with a list of fields and names we are going
to propose -- after we make sure the community does not have strong
objections. You can find this page here: http://websrv.cs.umt.edu/isis/index.php/CF_standard_names_for_Glaciology_and_Ice-Sheet_Modeling
  (Note that it is, as any wiki page, a work in progress.)

The biggest trouble we have is that existing glaciological standard
names have the land_ice prefix, suggesting that they correspond to
grounded ice. Some ice sheet models, however, model both grounded and
floating ice. Having to separate all the fields into grounded and
floating parts would not only be inconvenient, it would double the
storage space needed, which is no small issue.

Moreover, this distinction is somewhat confusing: floating_ice
includes sea_ice, which is chemically different from meteoric ice.

I would greatly appreciate any input, including clarifications of
intended uses of "land_ice", "floating_ice" and "sea_ice".

Thank you!
