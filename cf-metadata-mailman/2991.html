Dear all,

We are compiling a new dataset containing cloud physical properties
(CPP). CPP is an algorithm to derive cloud, precipitation and radiation
products from MSG satellite images. We are working on this dataset and
we would like to make it 100% CF compliant (if possible). We will make
the dataset publicly available by using OGC WMS, OGC WCS and FTP services.

For ten of the sixteen CPP products we found matching standard_names in
the CF standard name table. However, for the other six products we were
not able to find proper standard names.
Therefore we came up with some initial standard_names fore these products:
 - instrument_zenith_angle (degree): "Instrument zenith angle is the the
angle between the line of sight to the instrument and the local vertical" ;
 - solar_instrument_relative_azimuth_angle (degree): "The difference
between the solar azimuth angle and the instrument azimuth angle" ;
 - thermodynamic_particle_phase_at_cloud_top (status_flag): "The
thermodynamic phase of particles at the top of the cloud" ;
 - thermodynamic_particle_phase_at_cloud_top_infrared (status_flag):
"The thermodynamic phase of particles at the top of the cloud, measured
using only infrared radiances" ;
 - geometrical_thickness_of_liquid_water_cloud (m): "Cloud Geometrical
Thickness (cloud top height minus cloud base)" ;
 - effective_radius_of_cloud_condensed_water_particle_at_cloud_top (m):
"Particle effective radius for both liquid and ice water particles" ;

What is your opinion on these names? We used an online CF-1.4 checker,
it returned six errors, corresponding with the six standard_names listed
above.
Is there a better way do distuingish between
thermodynamic_particle_phase_at_cloud_top and
thermodynamic_particle_phase_at_cloud_top_infrared?

Thanks in advance,
Maarten
